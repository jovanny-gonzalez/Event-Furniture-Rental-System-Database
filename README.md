# Event-Furniture-Rental-System-Database
 Its purpose is to manage furniture reservations for events such as weddings, parties, and corporate gatherings.

## My Role: Database Design Lead
-Designed the ER Diagram and used MySQL forward engineering to generate the relational schema.

-Modeled one-to-many and many-to-many relationships using primary keys, foreign keys, and a bridge table.

-Designed table structures for customers, events, bookings, furniture inventory, and booking-to-furniture bridge records.

## Technologies/Concepts used
- MySQL/SQL
  
- ER DIAGRAM DESIGN
  
- MySQL forward engineering
  
- Relational database design
  
- Primary keys and foreign keys

- Bridge table implementation

- JDBC validation

## Project Features
- Customer, event, booking, and furniture inventory tables and a bridge table for the many-to-many relationship between bookings and furniture items
  
- Business rules for managing furniture reservations
  
- SQL query examples for analyzing customers, bookings, event types, and furniture rentals
  
- JDBC validation shown in the project presentation

## Content in Repository
- furniture_rental_system_dump (database schema and sample data)
  
- `Event-Furniture-Rental-System-Database.pdf` (project presentation with ER diagram, business rules, table examples, SQL examples, and JDBC output)
